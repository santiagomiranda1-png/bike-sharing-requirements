# Sistema de préstamo de bicicletas

## Stakeholders
1: administracion de la universidad.
2: estudiantes de la universidad.
3: equipo de mantenimiento de las bicicletas. 

## Requisitos funcionales

RF1: El sistema debe permitir a los estudiantes consultar la disponibilidad de bicicletas en el campus.

RF2: El sistema debe permitir a los estudiantes solicitar o reservar una bicicleta disponible.

RF3: El sistema debe registrar el préstamo de una bicicleta a un estudiante.

RF4: El sistema debe permitir registrar la devolución de una bicicleta.

RF5: El sistema debe permitir a los estudiantes reportar problemas o daños en una bicicleta.

RF6: El sistema debe mostrar el historial de préstamos de cada usuario.

RF7: El sistema debe permitir al personal de mantenimiento ver las bicicletas reportadas como dañadas.

RF8: El sistema debe enviar notificaciones cuando una bicicleta sea prestada o devuelta.
